发版说明

V2.5.0

【功能新增】
  • 该版本新增对4GB超大图的取流、渲染和存图。
    • 支持在OpenGL渲染模式下，渲染大于4GB的图像。您可以在您的程序中调用显示一帧图像扩展接口 MV_FG_DisplayOneFrameEx() 实现该功能。
      注解
      该功能需显卡支持。
    • 支持以TIFF和PNG格式，保存4GB以上大图。您可调用新增 MV_FG_SaveTiffToFileEx() 保存TIFF图像扩展接口和 MV_FG_SavePngToFileEx() 保存PNG图像扩展接口，分别通过 MV_FG_SAVE_TIFF_TO_FILE_INFO_EX 保存TIFF图像信息结构体和 MV_FG_SAVE_PNG_TO_FILE_INFO_EX 保存PNG图像信息结构体，在您的程序中实现对应功能。
    • 新增 MV_FG_INPUT_IMAGE_INFO_EX 输入的图像信息结构体，支持输入4GB以上大图。
  • 支持对取图去紫边，可去除取图中物体边缘由高反差、逆光等引起的紫边，从而提高图像质量。您可调用 MV_FG_PurpleFringing() 图像去紫边接口并设置 MV_FG_PURPLE_FRINGING_PARAM 去紫边结构体中的 nKernelSize 和 nEdgeThreshold 的值，调整去紫边的处理效果。

V2.4.0

【功能新增】
  • 新增导入设备属性扩展接口 MV_FG_FeatureLoadEx() 。
  • 新增错误信息结构体 MV_FG_NODE_ERROR 和错误信息列表结构体 MV_FG_NODE_ERROR_LIST 。
  • 新增版本不匹配状态码
【功能优化】
  • 增加对Ubuntu16.04-24.04 64bits和gcc/g++4.8.5及以上版本的兼容性
【框架调整】
  • 状态码章节并入至API参考章节，按照不同类型介绍。

V2.3.0

【功能新增】
  • 对齐Linux版本与Windows版本的功能，统一版本号为2.3.0
  • 增加CoaXPress/XoFLink/Camera Link/GigE Vision采集卡的多进程控制功能
  • 增加初始化SDK资源接口和释放SDK资源接口： MV_FG_Initialize() 、 MV_FG_Finalize()
  • 增加和完善C++示例程序
【功能优化】
  • 优化采集卡事件的性能和稳定性
  • 增加对Intel12代处理器的兼容
  • 增加对Ubuntu更高系统版本（22.04）和更高内核版本（6.1）的兼容性
  • 提升对部分第三方相机的兼容性
  • 提升对旧版本采集卡固件的兼容性
  • 修正遗留缺陷
【框架调整】
  • 优化编程引导章节，按照子功能点介绍调用流程并增加示例代码段

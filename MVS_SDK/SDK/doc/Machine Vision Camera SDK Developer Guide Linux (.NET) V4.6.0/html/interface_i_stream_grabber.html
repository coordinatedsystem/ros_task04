◆ SetImageNodeNum()

Int32 IStreamGrabber.SetImageNodeNum ( UInt32  num)

Sets number of nodes for SDK internal image buffer. The value is no less than 1, and this API should be called before calling IStreamGrabber.StartGrabbing().

Parameters
numIt refers to the number of buffer nodes.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Attention
Call this API before MV_CC_StartGrabbing() to set the number of nodes for SDK internal image buffer.
Note
It is one node by default.
Examples
Grab_Callback.cs, Grab_GetImageBuffer.cs, Grab_ImageClone.cs, Grab_Strategy.cs, Image_Recording.cs, InterfaceAndDevice.cs, LineScanSoftwareTrigger.cs, ParameterCamera_LineScanIOSettings.cs, ParameterInterface_SoftwareTrigger.cs, and ParametrizeCamera_AreaScanIOSettings.cs.

Int32 IStreamGrabber.ClearImageBuffer ( )

Clears the image buffer.

Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
  • This API supports clearing buffer for images not in use during image grabbing.
  • After switching to trigger mode from continuous mode, you can call this API to clear historical data buffer.

◆ StreamExceptionEvent

EventHandler<StreamExceptionEventArgs> IStreamGrabber.StreamExceptionEvent

Device stream exception event, only for USB devices.

Attention
It should be called before calling IStreamGrabber.StartGrabbing().

◆ StreamExceptionEventEx

EventHandler<StreamExceptionEventArgsEx> IStreamGrabber.StreamExceptionEventEx

Device stream exception event.

Attention
In the callback function registered via this API, IStreamGrabber.StopGrabbing() and IDevice.Close() cannot be called, and the callback function can only be used for message notification.
It does not support virtual devices and importing the third-party CTI files.

Camera Control

This chapter introduces how to call the SDK APIs to realize basic camera control functions such as enumeration and subsequent key functions such as parameter configuration, image acquisition, and image processing on your application.

Prerequisite

Completed the SDK initialization. See details in SDK Initialization.

Step 1 Enumerate Cameras

Call one of the following APIs as needed to enumerate cameras in the same IP segment.

  • Option 1: Call DeviceEnumerator.EnumDevices() (where the device interface type TLayerType needs to be configured). After calling succeeds, the device list devInfoList of the corresponding type will be returned.

    • When TLayerType is MvGigEDevice, all GigE devices including virtual GigE devices and GigE devices on frame grabbers will be enumerated.

    • When TLayerType is MvUsbDevice, all USB devices including virtual USB devices will be enumerated.

    • The TLayerType can also be MvGenTLCameraLinkDevice, MvGenTLCXPDevice, MvGenTLXoFDevice, etc. See details in DeviceTLayerType.

    Attention
    The memory of device list is internally allocated. When this API is called, the SDK will release and apply for the device list memory. Make sure to avoid enumeration operations of multiple threads.
    /*Enumerate cameras*/
    List<IDeviceInfo> devInfoList = null;
    int ret = DeviceEnumerator.EnumDevices(DeviceTLayerType.MvGigEDevice | DeviceTLayerType.MvUsbDevice | DeviceTLayerType.MvGenTLCXPDevice | DeviceTLayerType.MvGenTLXoFDevice, out devInfoList);
    if(ret == MvError.MV_OK)
    {
    /*Process camera information*/
    }

  • Option 2: For calling DeviceEnumerator.EnumDevicesEx() (comparing with the general enumeration API DeviceEnumerator.EnumDevices()), you can configure the manufacturer name manufacturerName to get the filtered enumeration result, which is the device list devInfoList corresponding to the manufacturer name.

    /*Enumerate cameras*/
    List<IDeviceInfo> devInfoList = null;
    int ret = DeviceEnumerator.EnumDevicesEx(DeviceTLayerType.MvGigEDevice | DeviceTLayerType.MvUsbDevice | DeviceTLayerType.MvGenTLCXPDevice | DeviceTLayerType.MvGenTLXoFDevice, "XXX", out devInfoList);
    if(ret == MvError.MV_OK)
    {
    /*Process cameras of manufacturer XXX*/
    }

  • Option 3: For calling DeviceEnumerator.EnumDevicesEx2() (comparing with the general enumeration API DeviceEnumerator.EnumDevices()), you can:

    • Filter the results by manufacturer name manufacturerName and get the returned device list devInfoList corresponding to the manufacturerName. If manufacturerName is null, the list of devices from all manufacturers will be returned.
    • Sort the devices from the returned device list, supporting multiple sorting methods, including serial No. and camera IP address.
    • Enumerate devices by TLayerType. If TLayerType is MV_GIGE_DEVICE, only GigE cameras in the same IP segment will be enumerated, excluding the virtual GigE cameras and GigE cameras connected to frame grabbers.
    /*Enumerate cameras*/
    List<IDeviceInfo> devInfoList = null;
    /*Only GigE cameras from the manufacturer XXX are enumerated and the enumeration results are sorted by camera serial No.*/
    int ret = DeviceEnumerator.EnumDevicesEx2(DeviceTLayerType.MvGigEDevice, SortMethod.SortBySerialNumber, "XXX", out devInfoList);
    if(ret == MvError.MV_OK)
    {
    /*Process GigE cameras from the manufacturer XXX*/
    }

Step 2 Create Camera Instance

Create camera instance via the following options.

  • Option 1: Create camera instance based on device information

    Call DeviceFactory.CreateDevice() (where configuring device information deviceInfo is required) to create the different camera instances by configured deviceInfo.

    Attention
    This API captures the exceptions sent by SDK via try…catch when creating instance fails.
    try
    {
    /*Create camera instance*/
    Idevice device = DeviceFactory.CreateDevice(deviceInfo);
    /*Turn on camera*/
    device.Open();
    /*If it is GigE camera*/
    if (device is IGigEDevice)
    {
    IGigEDevice gigEDevice = device as IGigEDevice;
    /*Configure the optimal packet size for GigE camera*/
    int packetSize;
    gigEDevice.GetOptimalPacketSize(out packetSize);
    gigEDevice.Parameters.SetIntValue("GevSCPSPacketSize", packetSize);
    }
    else if (device is IUSBDevice)
    {
    /*Set timeout duration for sync reading and writing of USB*/
    IUSBDevice usbDevice = device as IUSBDevice;
    usbDevice.SetSyncTimeOut(1000);
    }
    }
    catch(Exception e)
    {
    Console.WriteLine("Create device fail, {0}", e.Message);
    }

  • Option 2: Create camera instance based on IP address

    Call DeviceFactory.CreateDeviceByIp() (where configuring the device IP address deviceIp and network interface IP address netExportIp is required) to create camera instance.

    Attention
    • This API is only supported by GigE cameras.
    • This API captures the exceptions sent by SDK via try…catch when creating instance fails.
    /*Create camera instance*/
    try
    {
    Idevice device = DeviceFactory.CreateDeviceByIp(deviceIp, netExportIP);
    }
    catch(Exception e)
    {
    Console.WriteLine("Create device fail, {0}", e.Message);
    }
    // TODO: Turn on the camera, configure camera parameters, etc.

Step 3 Turn on the Camera

"Turn on the camera" refers to establishing the connection between the camera instance and physical camera, for the camera instance to gain access permission and establish communication with the physical camera.

  • Option 1: Call IDevice.Open() [1/2] to turn on the camera. In this way, the camera instance owns the exclusive access permission to the physical device.

    /*Turn on the camera*/
    int ret = device.Open();
    if(ret != MvError.MV_OK)
    {
    Console.WriteLine("Open device fail: {0}", ret);
    }
    else
    {
    Console.WriteLine("Open device success!");
    // TODO: Configure camera parameters, etc.
    }

  • Option 2: Call IDevice.Open() [2/2] to turn on the camera. Configuring the node AccessMode (access permission) and switchoverKey (secret key for switching access permission) is required for turning on device with specified access permission.

    Attention
    This API is only supported by GigE devices.
    /*Open camera*/
    int ret = device.Open(Accessmode, switchoverKey);
    if(ret != MvError.MV_OK)
    {
    Console.WriteLine("Open device fail: {0}", ret);
    }
    else
    {
    Console.WriteLine("Open device success!");
    // TODO: Configure camera parameters, etc.
    }

(Optional) Step 4 Acquire Images and Perform Other Operations as Needed

After the camera is turned on, you can:

  • Call SDK APIs to get camera events. See details in Event Acquisition.

  • Call SDK APIs to configure camera parameters. See details in Device Parameter Configuration.

  • Call SDK APIs to perform image acquisition related functions. See details in Image Acquisition.

    After image acquisition, you can perform further operations of image processing. See details in Image Processing

Step 5 Turn off the Camera

When the communication between camera instance and physical camera is no longer needed, call IDevice.Close() to turn off the camera.
/*Turn off the camera*/
int ret = device.Close();
if(ret != MvError.MV_OK)
{
Console.WriteLine("Close device fail: {0}", ret);
}
else
{
Console.WriteLine("Close device success!");
}

Step 6 Destroy Camera Instance

Call Dispose() to destroy camera instance and release the unmanaged resources occupied by camera instance.
Attention
After this API is called, the previous camera instance will be invalid and cannot be used for other APIs anymore.
/*Destroy camera instance*/
device.Dispose();
device = null;

Step 7 Destroy SDK Resources

Call SDKSystem.Finalize() to destroy SDK resources.

◆ DeviceAccessMode

enum DeviceAccessMode
strong

Device access modes, only supported by GigE devices.

Attention
MV_ACCESS_ExclusiveWithSwitch, MV_ACCESS_ControlWithSwitch, MV_ACCESS_ControlSwitchEnable, and MV_ACCESS_ControlSwitchEnableWithKey modes are not supported.
Enumerator
AccessExclusive 

Access Exclusive. The device access permission is exclusively occupied by the current process, and only the CCP register is allowed to be read by other processes.

AccessExclusiveWithSwitch 

For this mode, the device can be preemptively accessed by the current process from other processes under mode 5 ( AccessControlSwitchEnable ) and then turned on in the mode of access exclusive by the current process.

AccessControl 

Access Control. If device is accessed by the current process under this mode, all registers will be allowed to be read by other processes.

AccessControlWithSwitch 

For this mode, the device can be preemptively accessed by the current process from other processes under mode 5( AccessControlSwitchEnable ) and then turned on in the mode of access control by the current process.

AccessControlSwitchEnable 

For this mode, the device access permission of the current process can be preempted by other processes.

AccessControlSwitchEnableWithKey 

For this mode, the device can be preemptively accessed by the current process from other processes under mode 5 ( AccessControlSwitchEnable ). After access permission being preempted, the device access permission supports later being preempted by other processes.

AccessMonitor 

For this mode, the device can be accessed via AccessMonitor by the current process only if the device is previously occupied by other processes under mode 3 ( AccessControl ). If the device is previously occupied by other processes with access permission set as AccessExclusive, device accessing by the current process under AccessMonitor mode is not supported.

Detailed Description

Includes APIs and parameters of camera enumerations, including GigE vision camera, USB3 vision camera, and cameras on frame grabber (with GigE vision, camera link, CoaXPress, or XoFlink interface).

Examples
Events_Camera.cs, GigE_ForceIP.cs, Grab_ActionCommand.cs, Grab_Callback.cs, Grab_ChunkData.cs, Grab_GetImageBuffer.cs, Grab_GigEMultiCast.cs, Grab_ImageClone.cs, Grab_Strategy.cs, Image_Contrast.cs, Image_ConvertPixelType.cs, Image_HBDecode.cs, Image_Recording.cs, Image_Save.cs, LineScanSoftwareTrigger.cs, MultiLightCtrl.cs, MultiLightCtrl_ImageStitching.cs, ParameterCamera_FileAccess.cs, ParameterCamera_LineScanIOSettings.cs, ParameterCamera_LoadAndSave.cs, ParametrizeCamera_AreaScanIOSettings.cs, and Reconnect.cs.

◆ EnumDevices()

static Int32 DeviceEnumerator.EnumDevices ( DeviceTLayerType  TLayerType,
out List< IDeviceInfo >  devInfoList 
)
static

Enumerates devices.

Parameters
TLayerTypeIt refers to the interface types of device.
devInfoListIt refers to the device list.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
  • For device with MvGigEDevice interface, all GigE devices including virtual GigE devices and GigE devices on frame grabbers will be enumerated.
  • For device with MvUsbDevice interface, all USB devices including virtual USB devices will be enumerated.
Attention
The memory of device list is internally allocated. When this API is called, the SDK will release and apply for the device list memory. It is recommended to avoid multithreaded enumeration operations.
Later Calling
After device is enumerated, device instance can be created via DeviceFactory.
Examples
Events_Camera.cs, GigE_ForceIP.cs, Grab_ActionCommand.cs, Grab_Callback.cs, Grab_ChunkData.cs, Grab_GetImageBuffer.cs, Grab_GigEMultiCast.cs, Grab_ImageClone.cs, Grab_Strategy.cs, Image_Contrast.cs, Image_ConvertPixelType.cs, Image_HBDecode.cs, Image_Recording.cs, Image_Save.cs, LineScanSoftwareTrigger.cs, MultiLightCtrl.cs, MultiLightCtrl_ImageStitching.cs, ParameterCamera_FileAccess.cs, ParameterCamera_LineScanIOSettings.cs, ParameterCamera_LoadAndSave.cs, ParametrizeCamera_AreaScanIOSettings.cs, and Reconnect.cs.

Enumerates devices, supporting enumerating devices by specified specified sorting method and filtering by manufacturer name (extended API 2).

Parameters
TLayerTypeIt refers to the interface types of device.
sortMethodIt refers to sorting method.
manufacturerNameIt refers to the manufacturer name.
devInfoListIt refers to the device list.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
  • For device with MvGigEDevice interface, only GigE devices in specified IP segment will be enumerated, excluding virtual GigE devices and GigE devices on frame grabbers.
  • For device with MvUsbDevice interface, only general USB devices will be enumerated, excluding virtual USB devices.
Attention
The memory of device list is internally allocated. When this API is called, the SDK will release and apply for the device list memory. It is recommended to avoid multithreaded enumeration operations.
Later Calling

After device is enumerated, device instance can be created via DeviceFactory.

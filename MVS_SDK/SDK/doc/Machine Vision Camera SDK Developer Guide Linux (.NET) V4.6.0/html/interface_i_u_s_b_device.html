◆ SetTransferSize()

Int32 IUSBDevice.SetTransferSize ( UInt32  transferSize)

Sets transmission packet size of USB3 vision cameras.

Parameters
transferSizeIt refers to the size of the transmission packet (1 MB by default), unit: Byte, range: [0x400, 0x200000].
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Attention
Increasing the packet size can reduce the CPU usage, but for different computers and USB expansion cards, the compatibility is different. If the packet size is too large, image acquisition might fail.

◆ SetTransferWays()

Int32 IUSBDevice.SetTransferWays ( UInt32  transferWays)

Sets number of transmission channels of USB3 vision cameras.

Parameters
transferWaysIt refers to number of transmission channels, range: [1, 10].
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
This parameter can be adjusted based on computer performance, device image frame rate, device image size, and device memory usage. But compatibility differs due to different PC and USB expansion cards.

◆ GetTransferWays()

Int32 IUSBDevice.GetTransferWays ( out UInt32  transferWays)

Gets the number of transmission channels of USB3 vision cameras.

Parameters
transferWaysIt refers to the number of transmission channels.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
This API is used to get async image grabbing nodes of USB3 vision cameras. For an USB3 vision camera of 2000 W, the default async image grabbing nodes for MONO8 is 3, for YUV is 2, for RGB is 1, and for others is 8.

◆ SetEventNodeNum()

Int32 IUSBDevice.SetEventNodeNum ( UInt32  eventNodeNum)

Sets the number of event buffer nodes of USB3 vision cameras.

Parameters
eventNodeNumIt refers to the number of event buffer nodes, range: [1, 64].
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
Call this API to set the number of the buffer nodes for the current USB3 vision event. The default value is 5.

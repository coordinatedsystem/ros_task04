MV_CAMCTRL_API int __stdcall MV_CC_DisplayOneFrameEx ( IN void *  handle,
IN void *  hWnd,
IN MV_DISPLAY_FRAME_INFO_EX *  pstDisplayInfo 
)

Displays one frame of image (extended API 1).

Parameters
handle[IN] It refers to the device handle.
hWnd[IN] It refers to window handle.
pstDisplayInfo[IN] It refers to image information.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
 • The rendering mode can be set to GDI (default) or D3D. Once it is switched from GDI to D3D, it cannot be switched back to GDI.
  • The GDI mode is applicable to all computers with no requirements on graphic card performance.
  • The D3D mode is applicable to computers that has installed graphic card and its memory larger than 1 GB. In this mode, the live view image on the client will be better than that in GDI mode.
  • This API supports rendering images in pixel format. For PixelType_Gvsp_RGB8_Packed, PixelType_Gvsp_BGR8_Packed, and PixelType_Gvsp_Mono8, the API supports the width and height in integer type. For other pixel formats, the API only supports the width and height in short type.

MV_CAMCTRL_API int __stdcall MV_CC_DisplayOneFrameEx2 ( IN void *  handle,
IN void *  hWnd,
IN MV_CC_IMAGE *  pstImage,
unsigned int  enRenderMode 
)

Displays one frame of image (extended API 2).

Parameters
handle[IN] It refers to the device handle.
hWnd[IN] It refers to window handle.
pstImage[IN] It refers to the image information.
enRenderMode[IN] It refers to the image rendering mode. 0: OpenGL.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
 • It supports rendering an image over 4 GB in pixel formats of PixelType_Gvsp_RGB8_Packed, PixelType_Gvsp_BGR8_Packed, and PixelType_Gvsp_Mono8.
 • For images under 4 GB, the image width and image height can be converted to integer type data.
 • To use this API, you should enter the value of nImageLen in structure MV_CC_IMAGE.

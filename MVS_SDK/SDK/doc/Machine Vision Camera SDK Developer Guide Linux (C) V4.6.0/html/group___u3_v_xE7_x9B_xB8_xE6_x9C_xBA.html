Detailed Description

Includes APIs supported by USB3 vision cameras.

Function Documentation

◆ MV_USB_SetTransferSize()

MV_CAMCTRL_API int __stdcall MV_USB_SetTransferSize ( IN void *  handle,
IN unsigned int  nTransferSize 
)

Sets transmission packet size of USB3 vision cameras.

Parameters
handle[IN] It refers to the device handle.
nTransferSize[IN] It refers to the size of the transmission packet (unit: byte), and the default value is 1 MB (1,048,576 bytes). In Windows system, it is between 0x400 to 0x400000. In Linux system, it is between 0x400 to 0x200000.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
Increasing the packet size can reduce the CPU usage, but for different computers and USB expansion cards, the compatibility is different. If the packet size is too large, image acquisition might fail.

◆ MV_USB_SetTransferWays()

MV_CAMCTRL_API int __stdcall MV_USB_SetTransferWays ( IN void *  handle,
IN unsigned int  nTransferWays 
)

Sets the number of transmission channels of USB3 vision cameras.

Parameters
handle[IN] It refers to the device handle.
nTransferWays[IN] It refers to the number of transmission channels. It should be between 1 to 10.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
This parameter can be adjusted based on computer performance, device image frame rate, device image size, and device memory usage. But compatibility differs due to different PC and USB expansion cards.

◆ MV_USB_GetTransferWays()

MV_CAMCTRL_API int __stdcall MV_USB_GetTransferWays ( IN void *  handle,
unsigned int *  pnTransferWays 
)

Gets the number of transmission channels of USB3 vision cameras.

Parameters
handle[IN] It refers to the device handle.
pnTransferWays[IN][OUT] It refers to the pointer to the number of transmission channels.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
This API is used to get the current number of U3V asynchronous image acquisition nodes. For USB3 vision cameras, the number of transmission channels is closely related to the packet size corresponding to the pixel format, and it can be calculated based on the max. asynchronous registration length/packet size of pixel format.
For 2000W devices, the default number is: MONO8: 3; YUV: 2; RGB: 1; others: 8.

)

Sets the number of event buffer nodes of USB3 vision cameras.

Parameters
handle[IN] It refers to the device handle.
nEventNodeNum[IN] It refers to the number of event buffer nodes, range: [1, 64].

◆ MV_USB_SetSyncTimeOut()

MV_CAMCTRL_API int __stdcall MV_USB_SetSyncTimeOut ( IN void *  handle,
IN unsigned int  nMills 
)

Sets the timeout duration for sync reading and writing of USB3 vision devices (1000 ms by default), range: [1000, UNIT_MAX).

Parameters
handle[IN] It refers to the device handle.
nMills[IN] It refers to the timeout duration for sync reading and writing (1000 by default), unit: millisecond.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Note
Increasing the timeout duration for sync reading and writing can help deal with the problem that some cameras' parameter configuration process is very slow (more than 1000 ms).

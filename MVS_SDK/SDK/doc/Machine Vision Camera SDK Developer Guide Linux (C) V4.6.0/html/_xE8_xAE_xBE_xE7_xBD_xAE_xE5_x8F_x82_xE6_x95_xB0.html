Parameter Configuration

SDK can access and configure device parameters via GenICam Standard Protocol, and provides a set of parameter configuration APIs commonly used for cameras and frame grabbers to get and set parameter nodes of 6 types (integer, enumeration, float, boolean, string, and command).

This chapter includes:
  • Node Name and Node Type Search
  • Node Getting and Setting
  • Exclusive Configuration of GigE Cameras
  • Exclusive Configuration of U3V Cameras
  • Exclusive Configuration of Serial Port Devices

Node Name and Node Type Search

APIs of Node Getting and Setting nodes require configuring the node name and node type.
You can get the node name and node type via search on the client.

  1. Connect to the device via the client.

  2. Select a certain attribute parameter on the client attribute tree to view the parameter node information below the selected attribute.

Node Getting and Setting

Refer to the sample codes below to get and set node value in your program.
You can configure parameter nodes of 6 types: integer, enumeration, float, boolean, string, and command.

  • Integer Node:

    • Call MV_CC_GetIntValueEx() to get integer node.
    • Call MV_CC_SetIntValueEx() to set integer node.
    const char* strNodeName = "Width";
    // Get integer node
    MVCC_INTVALUE_EX stIntValue;
    int nRet = MV_CC_GetIntValueEx(handle, strNodeName ,&stIntValue);
    Check(nRet);
    // Set integer node
    nRet = MV_CC_SetIntValueEx(handle, strNodeName ,stIntValue.nCurValue);
    Check(nRet);


  • Enumeration Node:

    • Call MV_CC_GetEnumValue() or MV_CC_GetEnumValueEx() to get enumeration type node.
    • Call MV_CC_SetEnumValue() to set enumeration node.
    • Call MV_CC_GetEnumEntrySymbolic() to get the symbol of specified value the enumeration node.
    • Call MV_CC_SetEnumValueByString() to set enumeration attribute value.
    const char* strNodeName = "PixelFormat";
    MVCC_ENUMENTRY stEnumEntry;
    MVCC_ENUMVALUE stEnumValue;
    // Get enumeration type node
    nRet = MV_CC_GetEnumValue(handle, strNodeName, &stEnumValue);
    Check(nRet);
    // Get the symbol specified by the enumeration type node
    stEnumEntry.nValue = stEnumValue.nCurValue;
    nRet = MV_CC_GetEnumEntrySymbolic(handle, strNodeName, &stEnumEntry);
    Check(nRet);
    // Set enumeration type node
    nRet = MV_CC_SetEnumValue(handle, strNodeName ,stEnumValue.nCurValue);
    Check(nRet);
    // Set enumeration type attribute value
    const char* strValue = "Mono8"
    nRet = MV_CC_SetEnumValueByString (handle, strNodeName, strValue);
    Check(nRet);


  • Float Node:

    • Call MV_CC_GetFloatValue() to get float node.
    • Call MV_CC_SetFloatValue() to set float node.
    const char* strNodeName = "ExposureTime";
    // Get float node
    MVCC_FLOATVALUE FloatValue;
    int nRet = MV_CC_GetFloatValue(handle, strNodeName , &FloatValue);
    Check(nRet);
    // Set float node
    nRet = MV_CC_SetFloatValue(handle, strNodeName , FloatValue.fCurValue);
    Check(nRet);


  • Boolean Node:

    • Call MV_CC_GetBoolValue() to get boolean node.
    • Call MV_CC_SetBoolValue() to set boolean node.
    const char* strNodeName = "GammaEnable";
    // Get boolean node
    bool bValue = false;
    int nRet = MV_CC_GetBoolValue(handle, strNodeName , &bValue);
    Check(nRet);
    // Set boolean node
    nRet = MV_CC_SetBoolValue(handle, strNodeName , bValue);
    Check(nRet);


  • String Node:

    • Call MV_CC_GetStringValue() to get string node.
    • Call MV_CC_SetStringValue() to set string node.
    const char* strNodeName = "DeviceUserID";
    // Get string node
    MVCC_STRINGVALUE StringValue;
    int nRet = MV_CC_GetStringValue(handle, strNodeName , &StringValue);
    Check(nRet);
    // Set string node
    nRet = MV_CC_SetStringValue(handle, strNodeName , StringValue.chCurValue);
    Check(nRet);


  • Command Node: Call MV_CC_SetCommandValue() to set command node.

    const char* strNodeName ="UserSetLoad";
    // Set command node
    nRet = MV_CC_SetCommandValue (handle, strNodeName );
    Check(nRet);


Exclusive Configuration of GigE Cameras

If you are using GigE cameras, you can set optimal packet size and set resending packet.

  • Set Optimal Packet Size

    Before you start grabbing images after turning on the GigE camera, call MV_CC_GetOptimalPacketSize() to detect current network and get optimal communication packet size.

    You can configure the packet size of image grabbing with this optimal packet size to improve the transmission speed of image grabbing.

    // Get optimal packet size, and set it for GigE cameras
    if (stDeviceList.pDeviceInfo[nIndex]->nTLayerType == MV_GIGE_DEVICE)
    {
    int nPacketSize = MV_CC_GetOptimalPacketSize(handle);
    if (nPacketSize > 0)
    {
    // Configure the optimal packet size for GigE cameras
    nRet = MV_CC_SetIntValueEx(handle,"GevSCPSPacketSize",nPacketSize);
    Check(nRet);
    }
    }

  • Set Resending Packet

    GigE camera communicates data via network, so in some abnormal cases such as network congestion, data packet received from the SDK is incomplete. Resending command can control the camera to resend the incomplete data.

    Attention
    • Resending packets supported by the camera are available.
    • For occasional network congestion, you can address occasional packet loss through resending.
    • For continuous network congestion, it is recommended to troubleshoot the exception and optimize the network. Enabling resending packet directly may result in worse network congestion. Use this feature with caution.

    • You can refer to the following sample codes and call MV_GIGE_SetResend() to enable resending packet.

      unsigned int bEnable = true; // 1. Whether to enable resending packet
      unsigned int nMaxResendPercent = 10; // 2. The max. resending rate
      unsigned int nResendTimeout = 50; // 3. Resending timeout duration (unit: ms)
      nRet = MV_GIGE_SetResend(handle, bEnable, nMaxResendPercent, nResendTimeout);
      Check(nRet);
      1. bEnable: Whether to enable resending packet
      2. nMaxResendPercent: Configure the max. resending rate of the packet loss of the current frame. Do not resend the exceeding part to avoid worsening network congestion. When this value is set to 100, SDK will resend all packet loss of the current frame.
      3. nResendTimeout: Configure timeout duration of resending packet. Start counting time when the frame ends. If the counted time exceeds the set timeout, SDK will not request resending this packet loss data.

    • If the SDK does not receive data resent from the device, you need to control SDK to send the resending command again.

      The following sample codes show how to call MV_GIGE_SetResendTimeInterval() to set the interval between two resending requests for one resending packet and MV_GIGE_SetResendMaxRetryTimes() to set max. resending times.

      // Set the interval between two resending requests for one resending packet
      unsigned int nMillisec = 10;
      nRet = MV_GIGE_SetResendTimeInterval(handle, nMillisec);
      Check(nRet);
      // Set max. resending times
      unsigned int nRetryTimes = 20;
      nRet = MV_GIGE_SetResendMaxRetryTimes(handle, nRetryTimes);
      Check(nRet);
      • nMillisec: It refers to the interval between two resending requests for one resending packet (10 ms by default). The shorter the interval is, the more network resources occupied by the resending packet are, increasing the pressure of network and device.
      • nRetryTimes: 20 by default. It represents that for single packet loss data, SDK can request resending command for 20 times at most.


Exclusive Configuration of USB3 Vision Cameras

If you are using USB3 vision device, you can adjust control channel, and stream channel to address corresponding exception.

  • Control channel is used to pass control command. Abnormal control channel may result in slower execution of transmission control command. Therefore, the API times out and an exception will be returned.

    You can refer to the following sample code, and call MV_USB_SetSyncTimeOut() to adjust timeout duration of control channel, so as to be compatible with this type of exception.

    // Timeout duration (1000 ms by default)
    unsigned int nMills = 1000;
    // Set the timeout duration for sync reading and writing of USB3 vision devices.
    nRet = MV_USB_SetSyncTimeOut(handle,nMils);
    Check(nRet);

  • Stream channel is used to transmit image data. Insufficient bandwidth of stream channel or system resources may result in transmission failure and resource creation failure.

    You can refer to the following sample code, and call MV_USB_SetTransferSize() to adjust the packet size, or call MV_USB_SetTransferWays() to adjust the number of USB3 vision stream channels, so as to reduce resources occupied by SDK.

    // Set the max. image data packet size of USB3 vision camera
    unsigned int nTransferSize = 1*1024*1024; // 1 MB by default
    nRet = MV_USB_SetTransferSize(handle,nTransferSize);
    Check(nRet);
    // Set the number of buffers for USB3 vision stream channels of the SDK
    Unsigned int nTransferWay = 2;
    nRet = MV_USB_SetTransferWays(handle,nTransferWay);
    Check(nRet);


Exclusive Configuration of Serial Port Devices

Serial port device is connected via serial port, and the communication protocol complies with GenCP. For this type of device, you can call MV_CAMERALINK_DEVICE() for enumeration.
Note
Serial port devices include light controllers and camera link cameras.
When the device communicates through serial ports, you can change the baud rate to improve transmission speed. Details are as follows:
  1. Call MV_CAML_GetSupportBaudrates() to get the supported baud rate between the device and the PC.
  2. Call MV_CAML_SetDeviceBaudrate() to set the baud rate of the device.
// Get the supported baud rate between the device and the PC
unsigned int nBaudrateAblity = 0;
nRet = MV_CAML_GetSupportBaudrates(handle,&nBaudrateAblity);
Check(nRet);
// Check if it is supported to set the baud rate to 115200. If yes, set it to 115200
if (nBaudrateAblity & MV_CAML_BAUDRATE_115200)
{
nRet = MV_CAML_SetDeviceBaudrate(handle,MV_CAML_BAUDRATE_115200);
Check(nRet);
}

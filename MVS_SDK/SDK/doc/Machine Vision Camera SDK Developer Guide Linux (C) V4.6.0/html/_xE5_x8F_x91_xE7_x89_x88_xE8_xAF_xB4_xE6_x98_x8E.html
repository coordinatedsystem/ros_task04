Release Notes

V4.6.0_June/2025

Added contents related to stream exception callback function.

New Features
  • Supported PCIe board type light controller. You can call related APIs in Serial Port Device APIs Serial Port Device to turn on and control the serial port light controller.
  • Added an extended API for registering an image callback: MV_CC_RegisterImageCallBackEx2(). You can call MV_CC_FreeImageBuffer() outside the callback function to release buffer.
    Note
    See MV_CC_RegisterImageCallBackEx2() for more details on the API.
  • Added the stream exception callback function to report exception occurred when grabbing images.
    • Added an API for registering a stream exception callback for cameras connected to GEV or USB3 vision frame grabbers: MV_CC_RegisterStreamExceptionCallBack().
    • Added an enumeration for stream exception types: MV_CC_STREAM_EXCEPTION_TYPE .
    • Added a structure of stream exception callback information: MV_CC_STREAM_EXCEPTION_INFO .
  • Added serial port control related APIs in Serial Port Control APIs . You can call these APIs to control serial ports to realize communication with serial port device (such as liquid lens) connected to the camera.
    • MV_CC_SerialPort_Open() for opening serial port.
    • MV_CC_SerialPort_Read() for reading serial port data.
    • MV_CC_SerialPort_Write() for writing data to serial port.
    • MV_CC_SerialPort_ClearBuffer() for clearing acquired serial port data.
    • MV_CC_SerialPort_Close() for closing serial port.
  • Added Callback Function Definitions, including image callback function and exception callback.
  • Optimized the API MV_CC_ConvertPixelTypeEx() to support converting Bayer 10/12 format to Bayer 16 format.
    Note
    For Bayer 10 packed and Bayer 12 packed formats, it is required to first convert the image format to Bayer 10 and Bayer 12, and then convert to Bayer 16.
  • Added the following Generic Status Codes :
    • 0x80000018: Insufficient DDR buffer.
    • 0X80000019: Insufficient stream channels.
Sample Programs
  • Added a sample program for showing how to set frame trigger and enable image acquisition via software trigger for line scan cameras: LineScanSoftwareTrigger.cpp .
  • Edited the sample program ParametrizeCamera_LineScanIOSettings.cpp , and you can refer to it to ensure compatibility with the camera node of new version.
  • Edited the sample programs: ref ChunkData.cpp , Grab_Asynchronous.cpp .
Deprecations
  • Deprecated the API MV_USB_RegisterStreamExceptionCallBack(), and you can call MV_CC_RegisterStreamExceptionCallBack() instead.

V4.5.1_February/2025

Added APIs related to ISP configuration. You can call the corresponding API to process images by ISP algorithm in your program.

New Features
  • Supported passing ISP parameters and processing images by ISP algorithm. After configuring image processing parameters by the ISP tool and saving the configuration file, you can call MV_CC_SetISPConfig() and MV_CC_ISP_CONFIG_PARAM to load the configuration file to your program, and then call MV_CC_ISPProcess() to process images by ISP algorithm.

V4.5.0_November/2024

About Upgrade
Changed nImageBufLen of image information structure MV_CC_IMAGE to nImageLen. For normal execution of the program in this version of client, you should upgrade the above codes in your program.
New Features
  • Supported purple fringing correction feature.. Added an API MV_CC_PurpleFringing() and a corresponding structure MV_CC_PURPLE_FRINGING_PARAM for correcting purple fringing caused by high-contrast or backlight circumstances to enhance image quality. You can set the values of nKernelSize and nEdgeThreshold in MV_CC_PURPLE_FRINGING_PARAM to adjust the effect of purple fringing correction.
    AddedSpecified Error Codes of Purple Fringing Correction, range: 0x10405000 to 0x10405002, to indicates errors occurred during calling MV_CC_PurpleFringing(). These error codes help you troubleshoot corresponding issues.
    Note
    This feature is applicable to images in RGB and BGR formats.
  • Supported acquiring, rendering, and saving images over 4 GB.
    • Supported calling MV_CC_GetPayloadSize() to register a buffer over 4 GB.
    • Supported calling MV_CC_RegisterImageCallBackEx() and MV_CC_GetImageBuffer() to get an image over 4 GB.
    • Supported calling MV_CC_DisplayOneFrameEx2() to render an image over 4 GB.
      Note
      To execute rendering of an image over 4 GB, the OpenGL should be 2.0 or later.
    • Supported calling MV_CC_SaveImageToFileEx2() to save image over 4 GB in PNG or TIFF format.
Improvements
  • In frame information data structure MV_FRAME_OUT_INFO_EX , added a member SubImageList: It refers to the information of sub images in image buffer list for image stitching of frame grabbers, including pstSubImage (subimage information) and nAligning (alignment). In multi-light control scenes, when splitting images via the frame grabber, you can use SubImageList to get the subimages.
  • In frame information data structure MV_FRAME_OUT_INFO_EX , added UserPtr: It refers to the user-defined pointers for registering buffer, including pUser (user-defined pointer) and nAligning (alignment).
  • In Status Codes ,added 0x80000017: Image height exception, and incomplete image frame is dropped.
  • Added a console sample program: ParametrizeCamera_AreaScanIOSettings.cpp. You can refer to this demo for configuring trigger node IO of area scan cameras in your program.
Deprecations
  • Deprecated the sample program of ForceIP.cpp in Samples.
Structures Adjusted
  • Updated Programming Guide, and you can refer to it for using the SDK.
  • Updated FAQs.
